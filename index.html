Краткое описание языка разметки оно же пример.<\p>

Процессор этого языка должен давать на выходе HTML-код страницы. Ниже описаны примерные правила, как он это должен делать.<\p>

Последовательность из двух переводов строки, между которыми нет ничего кроме пробельных символов, начинает новый абзац. Каждый абзац должен быть заключен в теги \...\<\p>

Текст окруженный с двух сторон<\em> одинарными символами подчерка должен помещаться в тег \.<\p>

Любой символ можно экранировать, чтобы он не считался частью разметки. \_Вот это\_, не должно выделиться тегом \.<\p>

Двумя символами<\strong> — должен становиться жирным с помощью тега \.<\p>

Внутри

выделения \ может быть \<\strong> выделение<\p><\em>.<\p>

Текст окруженный одинарными _обратными_ кавычками<\code> (backticks) должен попадать в тег \, и к нему не нужно применять остальные правила форматирования. То есть слово "обратными" не должно выделиться \ внутри code.<\p>

Подчерки_внутри_текста__и__цифр_12_3 не считаются выделением и должны оставаться символами подчерка.<\p>

__непарные _символы не считаются `выделением.<\p>
